# メイン画面設計書

**作成日**: 2025-11-25  
**バージョン**: 1.0  
**Phase**: 03  
**画面ID**: main  
**ファイル名**: support_main.php

---

## 1. 概要

### 1.1 画面の目的
サポート報告書の一覧表示・検索・管理を行うシステムのメイン画面。

### 1.2 画面の役割
- サポート報告書の一覧表示
- 検索条件による絞り込み
- 新規登録・編集・削除への導線
- マスタ設定画面への導線
- Excel出力
- ログアウト

### 1.3 アクセス条件
- 認証済みユーザーのみアクセス可能
- 未認証の場合はログイン画面へリダイレクト

### 6.3 一覧取得クエリ（SEL_D_作業報告_検索）

```sql
SELECT 
    d.SEQNO,
    d.対応開始日時,
    d.対応終了日時,
    d.顧客コード,
    d.顧客担当者名,
    d.部門コード,
    d.商品コード1,
    d.商品コード2,
    d.商品コード3,
    d.作業担当コード,
    d.対応区分コード,
    d.報告内容,
    d.対応内容フラグ1,
    d.対応内容フラグ2,
    -- ... 対応内容フラグ1〜20
    t.氏名 AS 担当者名,
    k.顧客名,
    s1.商品名 AS 商品名1,
    s2.商品名 AS 商品名2,
    s3.商品名 AS 商品名3,
    ku.対応区分名
FROM D_作業報告 d
LEFT JOIN SQL_作業担当 t ON d.作業担当コード = t.担当者コード
LEFT JOIN SQL_顧客 k ON d.顧客コード = k.顧客コード
LEFT JOIN M_商品 s1 ON d.商品コード1 = s1.商品コード
LEFT JOIN M_商品 s2 ON d.商品コード2 = s2.商品コード
LEFT JOIN M_商品 s3 ON d.商品コード3 = s3.商品コード
LEFT JOIN M_対応区分 ku ON d.対応区分コード = ku.対応区分コード
WHERE d.削除日時 IS NULL
  AND (@作業担当コード IS NULL OR d.作業担当コード = @作業担当コード)
  AND (@顧客名 IS NULL OR k.顧客名 LIKE '%' + @顧客名 + '%')
  AND (@商品コード IS NULL OR d.商品コード1 = @商品コード 
       OR d.商品コード2 = @商品コード OR d.商品コード3 = @商品コード)
  AND (@対応区分コード IS NULL OR d.対応区分コード = @対応区分コード)
  AND (@対応日開始 IS NULL OR d.対応開始日時 >= @対応日開始)
  AND (@対応日終了 IS NULL OR d.対応開始日時 <= @対応日終了)
  AND (@キーワード IS NULL OR d.報告内容 LIKE '%' + @キーワード + '%')
ORDER BY d.対応開始日時 DESC
OFFSET @offset ROWS
FETCH NEXT @limit ROWS ONLY
```

## 7. Ajax通信

### 7.1 検索処理（support_ajax01.php）

**リクエスト**:
- Method: POST
- Content-Type: application/x-www-form-urlencoded

**リクエストパラメータ**:

| パラメータ名 | 型 | 説明 |
|-------------|-----|------|
| action | string | 'search' |
| tantou | int | 担当者コード |
| kokyaku | string | 顧客名 |
| shohin | int | 商品コード |
| kubun | int | 対応区分コード |
| date_from | string | 対応日（開始）YYYY-MM-DD |
| date_to | string | 対応日（終了）YYYY-MM-DD |
| keyword | string | キーワード |
| content[] | array | 対応内容項目コード配列 |
| page | int | ページ番号（1始まり） |
| limit | int | 1ページあたり件数 |

```javascript
        date_from: $('#search-date-from').val(),
        date_to: $('#search-date-to').val(),
        keyword: $('#search-keyword').val(),
        content: [],
        page: page || 1,
        limit: $('#page-size').val()
    };
    
    // 対応内容チェックボックス
    $('input[name="search_content"]:checked').each(function() {
        params.content.push($(this).val());
    });
    
    // バリデーション
    if (!validateSearchParams(params)) {
        return;
    }
    
    // ローディング表示
    showLoading();
    
    $.ajax({
        type: 'POST',
        url: 'support_ajax01.php',
        data: params,
        dataType: 'json',
        success: function(response) {
            hideLoading();
            if (response.status === 'success') {
                renderList(response);
            } else {
                showError(response.message);
            }
        },
        error: function() {
            hideLoading();
            showError('通信エラーが発生しました。');
        }
    });
}
```

```javascript
function renderList(response) {
    var html = '';
    var data = response.data;
    
    if (data.length === 0) {

            html += '<td><button class="btn-edit" data-seqno="' + row.seqno + '">編集</button></td>';
            html += '</tr>';
        }
    }
    
    $('#list-body').html(html);
    $('#total-count').text(response.total + '件');
    
    renderPaging(response);
}
```

## 13. 備考

### 13.1 パフォーマンス考慮

- 一覧取得はページングを使用（OFFSET-FETCH）
- 検索はAjaxで非同期実行
- 大量データ時は表示件数を制限（最大100件/ページ）

### 13.2 レスポンシブ対応のポイント

- PC: テーブル形式で全カラム表示
- Tablet: テーブル形式だが横スクロール対応
- SP: カード形式で縦並び表示

### 13.3 既存システムとの違い

| 項目 | Accessシステム | WEBシステム |
|------|--------------|------------|
| 検索 | その場で再クエリ | Ajax非同期検索 |
| ソート | カラムクリック | カラムクリック（Ajax） |
| ページング | なし（全件表示） | あり（50件/ページ） |
| レスポンシブ | なし | PC/Tablet/SP対応 |

### 13.4 テスト観点

- 検索条件の組み合わせテスト
- ページング動作テスト
- ソート動作テスト
