**作成日**: 2025-11-25  
**バージョン**: 1.0  
**Phase**: 03  
**画面ID**: main  
**ファイル名**: support_main.php

## 1. 概要

### 1.1 画面の目的
サポート報告書の一覧表示・検索・管理を行うシステムのメイン画面。

### 1.2 画面の役割
- サポート報告書の一覧表示
- 検索条件による絞り込み
- 新規登録・編集・削除への導線
- マスタ設定画面への導線
- Excel出力
- ログアウト

### 1.3 アクセス条件
- 認証済みユーザーのみアクセス可能
- 未認証の場合はログイン画面へリダイレクト

### 6.3 一覧取得クエリ（SEL_D_作業報告_検索）

```sql
SELECT 
    d.SEQNO,
    d.対応開始日時,
    d.対応終了日時,
    d.顧客コード,
    d.顧客担当者名,
    d.部門コード,
    d.商品コード1,
    d.商品コード2,
    d.商品コード3,
    d.作業担当コード,
    d.対応区分コード,
    d.報告内容,
    d.対応内容フラグ1,
    d.対応内容フラグ2,
    -- ... 対応内容フラグ1〜20
    t.氏名 AS 担当者名,
    k.顧客名,
    s1.商品名 AS 商品名1,
    s2.商品名 AS 商品名2,
    s3.商品名 AS 商品名3,
    ku.対応区分名
FROM D_作業報告 d
LEFT JOIN SQL_作業担当 t ON d.作業担当コード = t.担当者コード
LEFT JOIN SQL_顧客 k ON d.顧客コード = k.顧客コード
LEFT JOIN M_商品 s1 ON d.商品コード1 = s1.商品コード
LEFT JOIN M_商品 s2 ON d.商品コード2 = s2.商品コード
LEFT JOIN M_商品 s3 ON d.商品コード3 = s3.商品コード
LEFT JOIN M_対応区分 ku ON d.対応区分コード = ku.対応区分コード
WHERE d.削除日時 IS NULL
  AND (@作業担当コード IS NULL OR d.作業担当コード = @作業担当コード)
  AND (@顧客名 IS NULL OR k.顧客名 LIKE '%' + @顧客名 + '%')
  AND (@商品コード IS NULL OR d.商品コード1 = @商品コード 
       OR d.商品コード2 = @商品コード OR d.商品コード3 = @商品コード)
  AND (@対応区分コード IS NULL OR d.対応区分コード = @対応区分コード)
  AND (@対応日開始 IS NULL OR d.対応開始日時 >= @対応日開始)
  AND (@対応日終了 IS NULL OR d.対応開始日時 <= @対応日終了)
  AND (@キーワード IS NULL OR d.報告内容 LIKE '%' + @キーワード + '%')
ORDER BY d.対応開始日時 DESC
OFFSET @offset ROWS
FETCH NEXT @limit ROWS ONLY
```

## 7. Ajax通信

### 7.1 検索処理（support_ajax01.php）

**リクエスト**:
- Method: POST
- Content-Type: application/x-www-form-urlencoded

**リクエストパラメータ**:

| パラメータ名 | 型 | 説明 |
|-------------|-----|------|
| action | string | 'search' |
| bumon | int|null | 部門コード |
| tantou | int|null | 担当者コード |
| kokyaku | string | 顧客名 |
| shohin | int|null | 商品コード |
| kubun | int|null | 対応区分コード |
| date_from | string|null | 対応日（開始）YYYY-MM-DD |
| date_to | string|null | 対応日（終了）YYYY-MM-DD |
| keyword | string | キーワード |
| content[] | array | 対応内容項目コード配列 |
| page | int | ページ番号（1始まり） |
| limit | int | 1ページあたり件数（PAGE_SIZE_OPTIONSより） |
| sort_column | string | ソート対象（taiou_start/tantou_name/kokyaku_name/seqno） |
| sort_order | string | ソート順（ASC/DESC） |

## 10. JavaScript処理

### 10.1 初期化

- DOMロード完了時に以下を実行。
  - `loadData(currentPage)` で support_ajax01.php から一覧を取得し、セッション保持中の検索条件・ソート・表示件数を反映。
  - 報告内容セルのホバー表示を行う `initHoukokuTooltip()` を登録。
  - アコーディオンヘッダークリックで検索フォームの開閉とアイコンの開閉状態を切替。
  - 検索/クリア/表示件数変更/ページング/ソート/編集/新規入力/Excel出力/マスタ起動の各イベントをバインド。
  - 部門選択変更時に `filterTantouByBumon()` で担当者候補をフィルタリング。

### 10.2 主要関数

- `loadData(page)`: 検索条件・ソート・表示件数を収集し、`AppAjax.post` で support_ajax01.php を呼び出して一覧を再描画。
- `renderTable(responseData)`: 取得データから行HTMLを生成し、商品名結合・対応内容表示・報告内容の短縮/全文データ属性をセットし総件数を更新。
- `updatePaging()`: 現在ページと総ページ数に応じてページングボタンの活性/非活性とページ情報を更新。
- `updateSortIcons()`: ソート対象ヘッダに `sorted` を付与し、昇順/降順アイコンを切替。
- `exportExcel()`: 現在の検索条件をクエリストリング化し、`export_excel.php` へのダウンロード遷移を実行。
- `filterTantouByBumon(bumonCode)`: 部門コードに一致しない担当者オプションを一時的に非表示にし、選択値が除外された場合はクリア。
- `initHoukokuTooltip()`: 報告内容セルのマウスオーバーで全文ツールチップを表示し、離脱で非表示化。
- `openMasterModal(url, title)` / `closeMasterModal()`: 画面内モーダルとして各マスタ画面をiframeで開閉する。

## 13. 備考

### 13.1 パフォーマンス考慮

- 一覧取得はページングを使用（OFFSET-FETCH）
- 検索はAjaxで非同期実行
- 大量データ時は表示件数を制限（最大100件/ページ）

### 13.2 レスポンシブ対応のポイント

- PC: テーブル形式で全カラム表示
- Tablet: テーブル形式だが横スクロール対応
- SP: カード形式で縦並び表示

### 13.3 既存システムとの違い

| 項目 | Accessシステム | WEBシステム |
|------|--------------|------------|
| 検索 | その場で再クエリ | Ajax非同期検索 |
| ソート | カラムクリック | カラムクリック（Ajax） |
| ページング | なし（全件表示） | あり（50件/ページ） |
| レスポンシブ | なし | PC/Tablet/SP対応 |
