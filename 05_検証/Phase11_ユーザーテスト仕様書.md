## 3. テストシナリオ

### 3.1 基本業務フロー（一般ユーザー向け）

#### シナリオ A: 新規問い合わせ登録

**想定所要時間**: 5〜10 分

| ステップ | 操作内容                                   | 確認ポイント                         |
| -------- | ------------------------------------------ | ------------------------------------ |
| 1        | ログイン画面にアクセス                     | 画面が正常に表示されるか             |
| 2        | 担当者コードとパスワードを入力してログイン | ログインできるか                     |
| 3        | メイン画面で「新規」ボタンをクリック       | ダイアログが開くか                   |
| 4        | 顧客検索ボタンで顧客を検索・選択           | 顧客情報が反映されるか               |
| 5        | 対応日・時刻を入力                         | 日付ピッカー、時刻入力は使いやすいか |
| 6        | 商品を選択                                 | 商品選択は直感的か                   |
| 7        | 対応区分を選択                             | 選択しやすいか                       |
| 8        | 対応内容項目をチェック                     | チェックボックスの動作は正常か       |
| 9        | 報告内容を入力（定型文使用可）             | テキスト入力、定型文選択は便利か     |
| 10       | 保存ボタンをクリック                       | 正常に保存されるか                   |

#### シナリオ B: 既存報告の検索・編集

**想定所要時間**: 5 分

| ステップ | 操作内容                                     | 確認ポイント             |
| -------- | -------------------------------------------- | ------------------------ |
| 1        | 検索条件を展開（アコーディオン）             | 展開動作は分かりやすいか |
| 2        | 報告日範囲を指定して検索                     | 検索が正常に動作するか   |
| 3        | 一覧から編集したいデータの「編集」をクリック | 編集ダイアログが開くか   |
| 4        | 報告内容を修正して保存                       | 更新が正常にできるか     |
| 5        | 一覧を確認して更新内容が反映されているか     | データ反映を確認         |

#### シナリオ C: 検索と出力

**想定所要時間**: 3 分

| ステップ | 操作内容                        | 確認ポイント               |
| -------- | ------------------------------- | -------------------------- |
| 1        | 担当者で絞り込み検索            | 絞り込みが正常か           |
| 2        | ソート（報告日昇順/降順）を実行 | ソートが正しく動作するか   |
| 3        | Excel 出力ボタンをクリック      | CSV がダウンロードされるか |
| 4        | CSV を Excel で開いて確認       | 文字化けなく表示されるか   |

### 3.2 管理者向けシナリオ

#### シナリオ D: データ削除

**想定所要時間**: 3 分

| ステップ | 操作内容                         | 確認ポイント             |

## 6. テスト結果判定基準

### 6.1 合格条件

1. **必須条件**（全て満たすこと）

   - 致命的なバグがないこと
   - 全ての基本業務フローが完遂できること
   - データの登録・更新・検索が正常に動作すること

2. **推奨条件**（原則満たすこと）
   - 応答速度が基準値以内であること
   - 一般ユーザーが迷わず操作できること
   - 旧 Access 版と同等以上の操作性

### 6.2 判定区分

| 判定           | 基準                                         |
| -------------- | -------------------------------------------- |
| ○ 合格         | 必須条件全てクリア、推奨条件もおおむねクリア |
| △ 条件付き合格 | 必須条件クリア、推奨条件に一部未達あり       |
| × 不合格       | 必須条件に未達あり                           |
